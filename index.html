Course Topics

🌿
Version Control with Git
Branching strategies, rebasing, cherry-picking, merge conflict resolution, and maintaining a clean commit history.
🔀
Pull Request Workflow
Forking repositories, writing meaningful PRs, code review etiquette, and CI/CD integration gates.
⚖️
Open Source Licensing
Permissive vs copyleft licenses, GPL, MIT, Apache 2.0, Creative Commons, and choosing the right license.
🏛️
Project Governance
BDFL, meritocracy, foundations (Apache, Linux), codes of conduct, and decision-making in OSS communities.
🐛
Issue Tracking
Writing quality bug reports, triaging issues, labeling, milestones, and working with project maintainers.
📦
Package Ecosystems
npm, pip, Maven, crates.io — understanding dependency graphs, semantic versioning, and supply-chain security.
🤖
CI / CD Pipelines
Automating tests, linting, and deployment with GitHub Actions, automated security scanning, and release pipelines.
📝
Documentation
Writing READMEs, CONTRIBUTING guides, changelogs (Keep a Changelog), and API docs that attract contributors.
02

Contribution Workflow

01
Fork & Clone
Fork the upstream repository to your account, then clone your fork locally. Add the original as an upstream remote.
git remote add upstream https://github.com/org/repo.git
02
Create a Feature Branch
Never commit directly to main. Name branches descriptively using conventions like feat/, fix/, or docs/.
git checkout -b feat/add-dark-mode
03
Commit with Conventional Commits
Write structured commit messages so changelogs and release notes can be generated automatically.
# type(scope): short description
feat(ui): add dark mode toggle to navbar
fix(auth): handle expired JWT gracefully
docs: update CONTRIBUTING guide
04
Open a Pull Request
Push your branch and open a PR. Fill in the template: what, why, how to test. Link the relevant issue with Closes #42.
05
Review, Revise, Merge
Address reviewer feedback, squash fixup commits, and wait for CI to pass. A maintainer merges — your code is now part of the project! 🎉
